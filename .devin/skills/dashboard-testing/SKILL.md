# Dashboard Testing Skill

## When to Use

- After any change to `apps/lantern-garage/public/index.html`, `app.js`, or `server.js`
- After changes to `dashboard/index.html` (Netlify cloud operator dashboard)
- After adding/removing buttons, links, or system panels
- When auditing HTML prototypes served by the garage server

## Prerequisites

1. **Node.js 20+** installed
2. Repo cloned at the working directory
3. Dependencies installed: `(cd apps/lantern-garage && npm install)`

## Start the Garage Server

```bash
cd apps/lantern-garage && node server.js &
# Serves on http://127.0.0.1:4177
# Wait a few seconds for startup
curl -s http://127.0.0.1:4177/api/status | head -c 200
```

The garage server serves:
- Static HTML pages from `apps/lantern-garage/public/`
- Markdown rendering via `/view?path=<relative-path>`
- Repo files via `/repo/<path>` (serves from repo root)
- API endpoints at `/api/status`, `/api/command`, `/api/actions/*`

## Two Dashboards

### 1. Garage Dashboard (Local)
- **URL:** `http://127.0.0.1:4177/`
- **Source:** `apps/lantern-garage/public/index.html` + `app.js`
- **Contains:** Chat interface, 6 JS action buttons, 11 link buttons, 15+ system panels

### 2. Cloud Operator Dashboard (Netlify)
- **URL:** Production at `https://lantern-os-cloud.netlify.app` or PR preview at `https://deploy-preview-<N>--magical-rabanadas-2c70f4.netlify.app`
- **Source:** `dashboard/index.html`
- **Contains:** 10 nav panels (Dream Journal, Convergence, Batch Jobs, Health Check, Repositories, RAG/PDF Sync, Agent Fleet, Arc Reactor, Run Receipts, Held/Blocked)

## Test Procedure

### Garage JS Action Buttons

| Button | ID | What it does | Linux expectation |
|--------|-----|-------------|-------------------|
| Refresh | `#refresh` | Calls `/api/status`, updates panels | Works — shows "Status refreshed" in log |
| Run Check | `#runLoop` | Calls `/api/command` with `!converge` | Fails — needs PowerShell |
| Near 20m Paper Block | `#nearTermKalshiBlock` | Calls `/api/actions/kalshi-near-term-paper-block` | Fails — needs PowerShell |
| Sync Evidence | `#flatRagIngest` | Calls `/api/actions/flat-rag-ingest` | May work or show error |
| Live Refresh | `#autoUpdate` | Toggles 30s auto-refresh | Works — toggles aria-pressed |
| Open Controls | `#localControls` | Calls `/api/actions/local-controls` | Fails — needs PowerShell |

**Note:** Run Check, Near 20m Paper Block, and Open Controls rely on PowerShell scripts designed for Windows. On Linux test VMs, these will show error messages in the action log. This is expected behavior, not a bug.

## Devin Secrets Needed

No secrets required for dashboard testing. The garage server runs without authentication.

## Tips

- The garage server auto-creates `data/` directories on first run — this is normal
- Chat messages are stored locally in `data/conversations/` — they persist across server restarts
- The `/api/status` endpoint returns JSON with all panel data — useful for quick validation
- Netlify preview URLs follow the pattern `deploy-preview-<PR#>--magical-rabanadas-2c70f4.netlify.app`
- The Netlify site only serves `dashboard/` directory contents (see `netlify.toml`)
