# Lantern OS IP Disclosure Boundary

Updated: 2026-06-06
Status: private boundary draft
Audience: Alex, legal counsel, trusted reviewers
Disclosure: private by default. Do not publish this file into a public repo without review.

## Purpose

Lantern OS contains a large invention backlog, including a generated 207-claim packet and 200+ possible patentable ideas. Public portfolio work must demonstrate capability without disclosing patent-sensitive mechanisms.

## Default Rule

Public docs may describe:

- high-level architecture;
- product surfaces;
- sanitized workflows;
- evidence-gated engineering discipline;
- public-safe demos using synthetic data;
- broad categories of invention backlog.

Public docs must not disclose:

- claim charts;
- unpublished implementation mechanisms;
- novelty arguments;
- counsel strategy;
- private user data;
- raw personal dreams;
- source-specific IP evidence not already public;
- filing-priority-sensitive details.

## Public-Safe Phrase

Use:

> Maintained an evidence-indexed invention backlog across local-first AI memory, provenance, policy gates, symbolic interfaces, and orchestration systems.

Do not use publicly without review:

> I have 207 patent claims / 200+ patentable ideas, including [specific mechanisms].

## Strongest Candidate Categories to Track Privately

Based on current scan, the strongest patent-candidate categories appear to be Lantern-native infrastructure, not speculative AGI/medical/robotics claims.

Private category list:

1. Evidence-gated wallet / operating surface.
2. Proof receipt ledger.
3. Context/provenance database.
4. Hypothesis lifecycle engine.
5. Policy gate over proof receipts.
6. Useful-work queue / allocator.
7. Local-first AI memory and archive routing.
8. Synthesasia symbolic-emotive indexing.
9. CSF/CADD convergence archive mechanics.
10. Convergence IO governance and fallback runtime.

## Evidence Classes

| Evidence Class | Meaning | Public Handling |
|---|---|---|
| repo_verified | Local file/script/artifact exists and is inspectable. | Can be referenced if public-safe. |
| report_verified | Described in generated report with source pointers. | Summarize cautiously. |
| concept_only | Idea exists but lacks implementation evidence. | Keep private or label clearly. |
| patent_candidate | Possible IP candidate. | Private by default. |
| counsel_needed | Requires patent attorney / ownership / inventorship review. | Do not publish specifics. |

## Counsel Packet Prep

For each patent candidate, prepare:

- idea ID;
- title;
- earliest evidence date;
- source artifact;
- implemented evidence;
- public disclosure history;
- inventorship notes;
- prior art concerns;
- claim scope draft;
- narrowing strategy;
- business use case.

## Portfolio Boundary

The portfolio should prove:

- engineering judgment;
- system architecture;
- sensitive-data design;
- product maturity;
- AI governance awareness;
- working demos.

It should not publish the invention ledger.
