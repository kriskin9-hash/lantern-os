# Patch B — Explicit Command Router

**Priority:** P0
**Files:** `apps/lantern-garage/public/dream-chat.html`, `apps/lantern-garage/routes/dream.js`
**Problem:** Only `!debug` is special-cased; all other bang-commands fall through as ordinary chat text.
**Impact:** Command-grammar expectations drift silently; report/export/three-doors commands have no enforceable distinction.

## Change

### Client-side (`dream-chat.html`)

Add a shared `parseBangCommand()` utility and gate unsupported commands before sending:

```javascript
function parseBangCommand(input) {
  const m = String(input || "").trim().match(/^!(\S+)(?:\s+(.*))?$/);
  if (!m) return null;
  return { name: m[1].toLowerCase(), args: (m[2] || "").trim() };
}

// In sendMessage() before streaming:
const cmd = parseBangCommand(text);
if (cmd) {
  if (cmd.name === "debug") { /* existing toggle */ }
  else if (cmd.name === "three-doors") { /* set explicit 3-door mode */ }
  else if (cmd.name === "report")  { /* route to report or show unsupported */ }
  else if (cmd.name === "export")  { /* trigger export or show unsupported */ }
  else {
    appendSystemBubble(`Unknown command: !${cmd.name}. Try: !debug, !report, !export.`);
  }
  inputEl.value = "";
  inputEl.style.height = "auto";
  return;
}
```

### Server-side (`routes/dream.js`)

Mirror the same gate in the stream handler so malformed/unknown commands get a `400`:

```javascript
function parseBangCommand(input) {
  const m = String(input || "").trim().match(/^!(\S+)(?:\s+(.*))?$/);
  if (!m) return null;
  return { name: m[1].toLowerCase(), args: (m[2] || "").trim() };
}

// In POST /api/dream/chat and /api/dream/chat/stream:
const cmd = parseBangCommand(message);
if (cmd) {
  const supported = ["debug", "three-doors", "report", "export"];
  if (!supported.includes(cmd.name)) {
    return res.status(400).json({ error: "unsupported_command", command: cmd.name });
  }
}
```
