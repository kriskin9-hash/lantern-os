# Patch C — Prefer `.env.local` Over `.env` for Secrets

**Priority:** P1
**Files:** `apps/lantern-garage/routes/dream.js`, `apps/lantern-garage/server.js`
**Problem:** Provider settings are written directly to `.env` at repo root, increasing secret-sprawl risk.
**Impact:** Contradicts the repo's own security guidance; `.env` is more likely to be accidentally committed than `.env.local`.

## Change

### `apps/lantern-garage/routes/dream.js`

Change the save target from `.env` to `.env.local`:

```javascript
const envFilePath = path.join(repoRoot, ".env.local");
```

### `apps/lantern-garage/server.js`

Load `.env.local` first, then fall back to `.env`:

```javascript
const candidateEnvFiles = [
  path.resolve(__dirname, "..", "..", ".env.local"),
  path.resolve(__dirname, "..", "..", ".env"),
];
for (const envPath of candidateEnvFiles) {
  if (!fs.existsSync(envPath)) continue;
  fs.readFileSync(envPath, "utf8").split("\n").forEach((line) => {
    const m = line.match(/^([A-Z0-9_]+)\s*=\s*(.*)$/);
    if (m && !process.env[m[1]]) {
      process.env[m[1]] = m[2].replace(/^['"]/g, "").replace(/['"]$/g, "");
    }
  });
}
```

### `.gitignore` (verify)

Ensure both patterns exist:

```gitignore
.env
.env.local
```

## Verification

```bash
# After saving a provider key via the UI:
ls -la .env.local  # should exist
ls -la .env        # should NOT be created by the app
```
