Quick help

Common questions

What is Keystone, really?

An AI assistant that runs on your own machine, remembers what matters to you, and gets better from experience — never locked to a single AI model.

How do I use it?

Just open the chat and type like you're texting a friend. Ask a question, ask for help with code, or talk about your day.

Do I need to pay or sign up?

No. You can try the chat for free with no account. Adding your own API key unlocks faster, smarter replies — entirely optional.

Something's not working?

Check the Quick Start, browse the guides below, or report it on GitHub.

Research & theory

The math and method

Start with the local coder — Σ₀ Ouro Coder (the single source of truth) and its Continual-Training Loop — then the convergence math and method behind it.

🌀

Σ₀ Ouro Coder ?The local coding AI that runs on your own machine: a small model (Ouro) that "loops" to think harder on tough problems, fine-tuned on this repo's own past sessions.

The single source of truth for the local coder — then (Qwen) → now (Ouro-1.4B + Σ₀ QLoRA + adaptive Q-exit depth) plus the loop mechanism, served as a drop-in model the coder path uses transparently.

 ♻️

Σ₀ Continual-Training Loop ?The machinery that lets the local coding model improve from its own verified work: write code → run it to prove it works → train on the winners → keep the new model only if it scores higher on a coding exam.

The closed flywheel — harvest → execution-verify → train → eval → eval-gated promote. Two ground-truth gates (only green subprocesses train; only a measured pass@1 win promotes), kept offline by design.

 📊

Progress Report — Shareholder Edition ?An honest, evidence-tagged status of where Keystone — the bounded, self-improving agent — stands: what shipped, the real numbers, and the next milestones.

2026-06-19 — what shipped (token-budgeted memory #772), the real metrics (34% golden-set baseline), the honest ceiling, and the roadmap. Every claim evidence-tagged.

 📐

Σ₀ Collapse Certificate ?Σ₀ ("sigma-zero") is the project's core stability math. This document proves the reasoning loop stays grounded instead of spiralling into confident nonsense.

Lyapunov-contraction theorem (scoped to normal A), the Σ₀ trigger, and four ground-truth-verified experiments.

 📄

TeX source for the certificate — for typesetting and print.

Tesseract Convergence Loop ?The step-by-step method the system reasons with — tracking how confident it is ("Bayesian beliefs") in a 4-D status "cube" and keeping receipts for every claim.

The 20-step convergence method — 4D status cube, Bayesian beliefs, and receipts.

 🧭

Kernel model frontier ?A grounded comparison of the looped model Keystone runs (Ouro) against the current crop of small open models — scored on our own metrics, not generic leaderboards.

Ouro LoopLM vs the 2026 dense small-model frontier (Qwen3-4B, Phi-4-mini, Gemma 3n E2B) — primary-source-verified, with a benchmark plan instead of a pick.

 🌀

Convergence-exit spiral ?Instead of stopping when the model is "confident enough", keep looping until its internal state stops moving — a true fixed point. Falsifiable, with a shipped prototype.

Recasts Ouro's Q-exit as convergence-exit (loop until ‖Δh‖/‖h‖<ε), then loop→spiral→tesseract over the Status-Cube. Every claim tagged implemented / proven-conditional / hypothesis.

 🧩

Convergence Core spine ?The design for turning Keystone's already-built parts (memory, task, tool, record + the Σ₀ safety system) into one closed agent loop — and the single feedback wire that's still missing.

The four core objects, the six-stage loop, and the write-back closure in code (kernel.py, verify.py) — verified file-by-file, with the honest "durable agent, not AGI" ceiling.
