## Core Thesis

Founder wisdom is not founder infallibility.

Founder wisdom means:

```text
accountable priors + independent evidence + protected alternate views + explicit updates + proof-checked execution
```

The founder can be biased, close to the work, tired, overloaded, or wrong. The
system is allowed to trust founder synthesis only when the founder makes the
model inspectable and correctable.

## Required Decision Ladder

| Rung | Name | Allowed Action |
|---:|---|---|
| 0 | Signal only | No action; collect more data. |
| 1 | Private priors | Record independent estimates before discussion. |
| 2 | Evidence agreement | Separate facts from assumptions, projections, and wishes. |
| 3 | Hypothesis comparison | Run ACH or an equivalent alternatives table. |
| 4 | Structured consensus | Use Delphi, review pods, or careful review rounds while preserving alternate views. |
| 5 | Founder synthesis | Choose one reversible action and write the reason. |
| 6 | Proof checkpoint | External result validates, falsifies, or changes the claim. |
| 7 | Institutional memory | Commit posterior, receipt, and lesson to ledger/RAG. |

Consensus below rung 4 is conversation. Founder action below rung 5 is impulse.
Confidence above rung 6 must be earned by evidence.

## Founder Wisdom Operating Loop

| Step | Action | Output |
|---:|---|---|
| 1 | Frame the claim and decision under uncertainty. | Decision card |
| 2 | Collect private priors before discussion. | Prior ledger |
| 3 | Separate observed facts, assumptions, projections, and wishes. | Evidence table |
| 4 | Compare alternatives and disconfirmers. | ACH matrix |
| 5 | Run debate pods or careful review rounds. | Alternate-view log |
| 6 | Founder updates posterior and chooses reversible action. | Action gate |
| 7 | Ship test and measure external outcome. | Receipt |
| 8 | Record posterior, lesson, and next action. | World-model ledger |

No decision can call itself consensus unless steps 2 through 5 happened. No
decision can call itself wisdom unless steps 6 through 8 happened.

## Decision Card Template

Use this before high-variance moves.

| Field | Entry |
|---|---|
| Decision | What are we choosing? |
| Hypotheses | H1 / H2 / H3 |
| Private priors | Founder / agent / advisor / customer |
| Evidence classes | Observed / source-cited / operator assertion / projection |
| Strongest disconfirmer | What would prove us wrong? |
| Alternate views | Who saw it differently and why? |
| Founder posterior | Updated probability with reason. |
| Action gate | If X then Y, otherwise hold. |
| Receipt | Link, screenshot, commit, invoice, user feedback. |

The founder may choose a different path than the group only by writing the
reason and the evidence that would change the decision.

## Confidence Rules

| Evidence Event | Allowed Score Effect |
|---|---|
| Repo/file/report created and validated | Small increase if verified. |
| Specific user or stakeholder feedback | Medium increase. |
| Paid invoice sent | Medium increase. |
| Payment received | Large increase. |
| Validation failure found | Short-term decrease; long-term increase only if fixed. |
| Consensus with no external proof | No increase. |
| Founder conviction only | No increase. |

Confidence can increase only from external evidence, successful prediction,
validated delivery, or a logged contradiction being resolved.
