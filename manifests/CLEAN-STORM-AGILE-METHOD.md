# Clean Storm Agile Method

Status: active sprint method.

Purpose: repeat a fast 12-step convergence loop that hammers the repo, trims
fat, validates evidence, and ships clean increments.

## Loop

| Step | Name | Output |
|---:|---|---|
| 1 | Status | branch, dirty state |
| 2 | Fetch | local/remote delta |
| 3 | Scan | stale/fake/TODO/duplicate candidates |
| 4 | Sort | first 2-4 actionable issues |
| 5 | Strike | narrow fixes |
| 6 | Trim | removed stale/fat surface |
| 7 | Tighten | boundaries kept only where useful |
| 8 | Bayes | priors/posteriors updated |
| 9 | Validate | targeted checks |
| 10 | Re-scan | fixed issues gone |
| 11 | Record | manifest or belief ledger update |
| 12 | Ship/Repeat | push, clean repo, next loop |

## Active Sprint Target

Current sprint converges the Super Jarvis Lantern OS top skill, flat RAG
dollhouse, literal COMET LEAP PDF/image assets, Archive.org / Wayback / commons
metadata batch lane, validation, and push.

## Fat To Trim First

- Fake workflow placeholders.
- Stale claims that a repo, remote, release, or asset state differs from
  observed state.
- Duplicated method descriptions when a skill now owns that method.
- Public-media download plans without rights state.

## Fat Not To Trim

- Boundary notes for physical disk work, phone boot, public-media rights, and
  v1.0.0 release approval.
- Asset manifests and hashes.
- Source paths for copied PDFs/images.
