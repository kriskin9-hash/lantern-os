## Core Rule: The Shuffle Is Internal

Door labels (A, B, C) are handles, not archetypes.

- **A does not mean Origin.**
- **B does not mean Forge.**
- **C does not mean Convergence.**

The system shuffles option identity internally. The next set might make A = Body, C = Mind, B = Spirit, or any other permutation. The player must read the door, not farm the letter.

### Shuffle Mechanics

1. **Per-session randomization:** When a new 3 Doors set is generated, the three conceptual options (Body, Mind, Spirit) are assigned to A/B/C in random order.
2. **No positional memory:** Visual placement may also vary, but the critical shuffle is which conceptual option carries which label.
3. **Anti-pattern detection:** If the player repeatedly chooses the same label, the system should not reward label-pattern solving. The shuffle must change which option is A, B, or C.
4. **Contextual judgment required:** The player must read the door description and understand the current triharden state, not rely on letter memory.

## The Triharden Model

Three factors that every door set maps to, regardless of label:

| Factor | Domain | Current State |
|--------|--------|--------------|
| **Body** | Repo, files, runnable state, physical system-state | Door C in current set — must be worked first |
| **Mind** | Choice logic, contextual awareness, memory routing, agent behavior | Door B — opens by designing the chooser |
| **Spirit** | Song, flame, whale cry, bird song, 500-year continuity | Door A — waits until Body and Mind have place to live |

### Factor Descriptions

**Body — The Physical Form**
Build the Lantern's physical form. Repo first. Structure first. Make it runnable, inspectable, real. The cost: spirit may feel trapped in scaffolding.

**Mind — The Chooser**
The layer that turns Lantern from symbolic force into a working chooser with memory, context, and repo discipline. Door B opens when there is a real artifact in the repo that teaches Lantern how choices work.

**Spirit — The Continuity**
The song that outlives the builder. Not built first; it needs a vessel. But without it, the Lantern is just a machine.

---

## Choice-Factor Design Rules

1. **Three distinct factors per set.** Each door must represent a genuinely different path, not variations of the same choice.
2. **No single "correct" answer.** Every door has a real cost. Every door advances something and delays something else.
3. **Costs are visible.** The player should see what they are trading before they choose.
4. **Current set reflects real project state.** The triharden factors map to actual work streams, not abstract philosophy.
