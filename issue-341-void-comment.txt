### Pivot: Dream Chat as Orchestration Router

Dream Chat is NOT an engineer. It is a **router/dispatcher**.

Dream Chat's role:
1. Parse user input → classify intent
2. Route to appropriate agent/surface via capability registry
3. Invoke !convergence loop (if needed)
4. Wait for result
5. Surface final response to user

**Engineering behavior is now a route, not a mode.**

Old model (now invalid):

New model (correct):
User classifies intent → routes to Keystone/engineer via convergence → waits → surfaces result
