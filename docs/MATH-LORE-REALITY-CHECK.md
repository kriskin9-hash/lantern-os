## Classification Labels

| Label | Meaning |
|---|---|
| Implemented | Exists in code or docs with testable behavior. |
| Prototype | Partial implementation exists; limitations remain. |
| Conceptual | Useful theoretical anchor, not implemented as formal machinery. |
| Lore / Metaphor | Creative symbolic language; not a literal scientific or engineering claim. |
| Private IP | Potentially sensitive claim area; do not publish mechanics without review. |

## Public Claim Hygiene

Use:

> Lantern OS uses math-inspired and evidence-gated design patterns: ternary addressing, sparse state maps, provenance records, and structured claim hygiene.

Avoid:

> Lantern OS proves observer-dependent reality or implements literal time navigation.

Use:

> The tesseract is a design metaphor and sparse navigation model.

Avoid:

> The tesseract physically contains all past and future states.

Use:

> Bayesian and Free Energy concepts inform the system's evidence discipline.

Avoid:

> The current system performs formal active inference unless the generative model is implemented.
