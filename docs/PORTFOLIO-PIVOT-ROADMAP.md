## Public Case Study Outline

### 1. Problem

Personal writing is sensitive, ambiguous, hard to search, and difficult to use safely with AI systems.

### 2. Solution

Build a local-first symbolic memory interface that captures user entries, classifies data, routes AI calls through capability and authority gates, falls back safely, and records provenance.

### 3. Architecture

User input -> DCF classification -> NAP denial check -> CCF capability check -> PCSF provider routing -> LLM/offline response -> AAPF provenance -> CSF/CADD export.

### 4. Demo

Use synthetic dreams only.

Example synthetic input:

> I dreamed I was in an old school with endless hallways. A blue door kept appearing, but each time I opened it I returned to the same room.

Demo output:

- structured entry;
- tags;
- symbols;
- emotional tone;
- Convergence IO route trace;
- provenance record;
- CSF/CADD export sample.

### 5. Engineering Notes

Show:

- provider fallback behavior;
- authority gates;
- data classification propagation;
- command normalization for legacy `!convergance` -> `!convergence`;
- health endpoint snapshot;
- privacy boundary.

### 6. Honesty / Scope Discipline

Do not claim:

- literal time travel;
- operational AGI;
- legal/medical/financial authority;
- implemented probabilistic Bayesian machinery if only conceptually referenced;
- fully image-first Three Doors unless implemented and tested;
- public release of private dream data;
- patentability without legal review.
