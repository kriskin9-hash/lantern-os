# Lantern OS Drive Agent Map

This document mirrors the Google Drive `AGENTS.md` map for Lantern OS agents. It exists to reduce token burn when agents need to work across the repo and Drive archive.

## Source split

GitHub is the source of truth for code, tests, repo docs, manifests, and implementation state.

Google Drive is the working/archive surface for operator-facing docs, CSF/CADD ingests, design records, exported briefs, PDFs, and non-code archival material.

## First-pass workflow

1. Read repo `AGENTS.md` first.
2. Read this document before broad Drive searches.
3. Search Drive with short targeted terms: `Lantern OS`, `CSF`, `CADD`, `Raven Door`, `Elephant Door`, `Xenon`, `Orion`, `convergence`, `archive`.
4. Prefer canonical titles and current dated ingests over duplicate exports.
5. Do not treat generated concept art as repo-derived unless repo/spec evidence was checked first.
6. Do not claim memory is proof. Memory is context until grounded by repo, Drive, or explicit operator statement.

## Convergence definition

`!convergence` means getting the most bang for your buck when you are in a van: choose the smallest bounded action that gives the largest useful result under limits of time, energy, tools, battery, connectivity, money, attention, and privacy.

Convergence is not perfection. It is a constrained operating loop: inspect what is real, pick the highest-leverage reversible move, validate it cheaply, record the receipt, then promote, hold, or archive.

## Drive document naming

Use these title prefixes when creating or sorting records:

- `CSF-INGEST-*` for canonical memory/context ingests.
- `CADD-*` for Capture, Assess, Distill, Dock records.
- `LANTERN-OS-*` for general operating docs.
- `RAVEN-DOOR-*`, `ELEPHANT-DOOR-*`, `XENON-*`, `ORION-*` for symbolic/product lanes.
- `ARCHIVE-*` for held, deprecated, duplicate, or historical material.

## Current connector limitation

As of this update, the available Google Drive connector can create, read, search, and edit document content, but I did not have a confirmed Drive metadata move/update action for changing parent folders. File movement into the new archival folder should be done manually or by a tool/runtime with parent-folder update support.
