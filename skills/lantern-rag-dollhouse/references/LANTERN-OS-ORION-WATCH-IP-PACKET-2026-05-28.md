## Official-source anchors

- USPTO provisional applications: a provisional can be filed without formal claims, oath/declaration, or IDS; it provides an early effective filing date for a later nonprovisional and permits use of `patent pending`; the provisional pendency is 12 months and cannot be extended.
- USPTO design patents: design patents protect the visual ornamental characteristics of an article of manufacture; utility patents protect the way an article is used and works; both design and utility patents may apply when both functional and ornamental invention are present.
- USPTO trademark process: an examining attorney reviews applications, searches for conflicting marks, examines specimens, and intent-to-use applications require later use in commerce and statement-of-use steps after notice of allowance.

## Converged IP strategy

File paths should separate three lanes:

1. Utility patent provisional: functional systems and methods.
2. Design patent drawings: ornamental smartwatch, bezel, watch face, strap, charger, and ship kit appearance.
3. Trademark / brand clearance: names, slogans, logos, product marks, and service marks.

Best first action: make a provisional invention disclosure packet with the image, functional descriptions, diagrams, flow charts, alternative embodiments, and named human inventor notes. Then run a professional prior-art search before filing nonprovisional claims.

## Candidate utility inventions

### U1 - On-wrist MK1 Reactor State Activation

Working title: `Wearable reactor-state activation based on wrist-presence confidence`.

Core concept: a smartwatch operating mode where a full MK1 reactor state becomes available only when the watch detects valid wrist presence through a combined presence signal, such as capacitive sensing, bioimpedance, optical pulse/PPG signal, motion continuity, skin contact, and secure boot state.

Possible novelty angle: not just wrist detection, but a layered confidence gate that unlocks a personalized high-output visual/audio/voice state only when the device is physically worn by the owner or authorized operator.

Draft independent claim concept:

A wearable computing device comprising a display, one or more biometric or contact sensors, a light-emitting bezel system, and a processor configured to determine a wrist-presence confidence state from multiple sensor signals; transition the device from a base state to an MK1 reactor state when the confidence state satisfies a threshold; and activate a corresponding display brightness, bezel-light pattern, haptic confirmation, and voice-control availability while the MK1 reactor state remains valid.

Dependent concepts:

- Confidence combines capacitive, bioimpedance, optical, motion, and temperature signals.
- Full brightness or full visual reactor state is disabled off-wrist.
- A local-only Lantern OS policy state controls which actions are allowed in the active mode.
- A haptic confirmation occurs when MK1 state becomes active.
- Removal from wrist drops back to base state and records a local state-change event.

### U2 - Reactor Sync Indicator and Personal Presence Ledger

Working title: `On-device state ledger for personal wearable activation`.

Core concept: the watch maintains a local state record of reactor sync events: inactive, on-wrist candidate, active, held, charging, ship mode, and local-only restricted. The user sees a reactor sync panel rather than raw engineering logs.

Possible novelty angle: converting device-trust state into a readable personal reactor metaphor that maps to actual hardware state and policy restrictions.

Draft claim concept:

A method for operating a wearable device comprising receiving wrist-presence sensor data, determining a wearable trust state, mapping the trust state to a user-facing reactor sync state, displaying the reactor sync state through a circular display motif, and restricting selected operating-system functions when the trust state is below a threshold.

### U3 - Orion Constellation Watch Face With 12-Stone Matrix State Encoding

Working title: `Constellation-based wearable state display with twelve-marker activation ring`.

Core concept: an Orion starfield watch face paired with a 12-marker ring. Three highlighted markers at 12, 4, and 8 encode the active triad. The star pattern provides emotional personalization while the ring markers encode device state.

Possible novelty angle: coupling an ornamental constellation face with a functional state indicator in a repeated 12-marker matrix and active triad.

Draft claim concept:

A graphical user interface for a wearable device comprising a constellation field, a time display, a twelve-position circumferential status array, and three simultaneously highlighted positions corresponding to an active device state, wherein the highlighted positions indicate an activation triad and the constellation field remains visible as a background state surface.

Note: GUI and watch-face patentability is difficult and must be assessed carefully. This lane may be better as design patent / copyright / trade dress unless a strong functional interaction is documented.

### U4 - Optical Self-Loop Bezel Light Pipe for Wearable State Verification

Working title: `Circular light pipe status loop for wrist-worn device`.

Core concept: a coded light pulse travels through a circular bezel light pipe and returns to a sensor region. The return is used to confirm light-path integrity, mode activation, or UI state.

Possible novelty angle: adapting the Lantern Ring optical self-loop concept to a smartwatch bezel as both visual effect and state verification.

Draft claim concept:

A wrist-worn device comprising a circular light pipe disposed around a display, at least one emitter configured to introduce a coded light pulse into the light pipe, at least one sensor positioned to detect the coded light pulse after traversal of the light pipe, and a controller configured to compare the detected pulse to an expected pulse pattern and alter a device state or visual indication based on the comparison.

Dependent concepts:

- Sensor at or near 12 o'clock.
- Active markers at 12/4/8 illuminate when the coded loop verifies.
- Failure to verify triggers a held or dimmed state.
- Light pipe communicates with haptic or audio confirmation.

### U5 - Push-to-Talk Crown With Voice Gate and Haptic Confirmation

Working title: `Wearable crown voice-gate with off-ring/off-watch audio route`.

Core concept: rotating/push crown activates a voice gate, using local microphone, Bluetooth microphone, or paired audio route. Haptic feedback confirms open/closed voice state.

Possible novelty angle: a physically obvious voice gate that maps to privacy-preserving OS state and rejects unattended/background activation unless user action and wrist-presence confidence are met.

Draft claim concept:

A wearable computing device comprising a crown control, a microphone input, a haptic actuator, and a wireless audio interface; wherein depression of the crown control activates a voice gate only when a wrist-presence state satisfies a threshold, and wherein the device provides haptic feedback indicating activation and deactivation of the voice gate.

### U6 - MK1 Ship Kit and Safe Pairing Ritual

Working title: `Packaged wearable onboarding kit with local-first activation sequence`.

Core concept: a product package containing watch, magnetic charger, quick-start card, and local-first pairing instructions that route first activation through NFC/Bluetooth and a privacy/safety checklist.

Possible novelty angle: likely weak utility patent, stronger design/trade dress/brand lane unless the activation ritual has a technical method beyond packaging.

## Prior-art / novelty risk map

High prior-art density:

- Smartwatches with wrist detection.
- Heart-rate/SpO2 sensors.
- NFC pairing/payment.
- Bluetooth audio.
- Push-to-talk controls.
- Haptic confirmation.
- Always-on AMOLED displays.
- Magnetic pogo charging.
- Premium watch cases and straps.

Better novelty candidates:

- Multi-sensor wrist-presence confidence specifically gating a named high-output `MK1 reactor` mode.
- Circular optical self-loop through a watch bezel light pipe with return sensing.
- Twelve-position active triad encoding tied to device state and wrist-presence confirmation.
- Local-first voice-gate policy requiring both physical control and wrist-presence state.
- Combined technical state transition: base -> candidate -> active -> held -> charging/ship mode, presented as Reactor Sync.

Weakest patent candidates:

- Pure Orion visual motif alone.
- Generic 46 mm round watch with strap.
- Generic magnetic charger and ship kit.
- Generic Bluetooth/NFC/heart-rate/haptic features.
